Langkah-Langkah Menggunakan Github
1. Melakukan sign-up di github.com
2. Membuat repository dengan cara mengklik pojok kanan atas (di sebelah ikon profile, tanda +) lalu pilih new repository
3. Beri nama pada repository baru, beri sedikit deskripsi dan tandai centang di kolom initialize this repository with a README
4. Untuk membuat branch, di repository klik drop-down yang ada di atas file list yang bertuliskan branch:master. Ketikkan nama branchnya di dalam textbox lalu pilih box biru yang bertuliskan create branch atau langsung tekan enter.
5. Membuat dan meng-commit perubahan dengan cara mengklik file README.md, setelah itu mengklik ikon pensil di pojok kanan atas untuk mengedit. Lalu tulis di editor apa yang mau ditulis. Lalu menuliskan commit message yang mendeskrpsikan perubahan yang dilakukan lalu klik tombol commit changes.
6. Untuk membuka pull request, klik tab pull request lalu di page tersebut klik new pull request. Di box example comparasions, pilih branch yang telah dibuat untuk dicompare dengan branch master. Lihat kembali perubahan di compare page dan pastikan itu yang akan disubmit, lalu klik create pull request.
7. Untuk menggabungkan hasil dari branch ke branch master, klik merge pull request. Lalu klik confirm merge. Lalu delete branchnya karena sudah tidak diperlukan lagi.
Menginisialisasi Git : git init
Mengecek status : git status
Memberitahu Git untuk memulai tracking perubahan dalam text : git add octocat.txt
Menyimpan staged changes menggunakan commit : git commit -m "Add cute octocat story"
Meng-add file dalam jumlah banyak ke direktori : git add '*.txt'
Commiting all changes : git commit -m 'Add all the octocat txt files'
Jurnal yang mengingat perubahan secara sistematis : git log
Mengecek perubahan dalam github : git pull origin master
Meletakkan perubahan saat kita sudah siap : git push -u origin master
Melihat perbedaan antara perubahan sebelumnya : git diff HEAD
Menambah file txt baru ke folder : git add octocat.txt
Menampilkan perubahan di stage area yang akan masuk ke perintah selanjutnya : git diff -staged
Menghapus file yang unstaged : git reset octocat.txt
Mengembalikan perintah terakhir : git checkout -octocat.txt
Menghapus branch file : git branch clean-up
Switch branch-branch file : git checkout clean-up
Menghapus file yang sudah ada dalam staging area : git rm '*.txt'
Melakukan commit pada perubahan yang sudah dilakukan : git commit -m "remove all the cats"
Pindah ke branch master : git checkout master
Menggabungkan branch clean_up ke branch master : git merge clean_up
Menghapus branch clean_up : git merge -d clean_up
Memasukkan perubahan yang sudah dilakukan ke remote repository : git push
